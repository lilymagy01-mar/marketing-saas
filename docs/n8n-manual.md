# 🛡️ LilyMag 사령부 운영 매뉴얼 (n8n v4.0 Final)

이 문서는 사장님의 마케팅 자동화 엔진을 단 한 번의 설정만으로 영구 가동하기 위한 최종 지침서입니다.

## 1. 개요
본 엔진은 **"1 Trigger - 7 Channels - 3 Media types"**를 지향하는 올인원 마케팅 폭격 시스템입니다. 사장님은 앱에서 '승인' 버튼만 누르면 되고, 나머지 모든 복잡한 로직은 이 엔진이 처리합니다.

## 2. 핵심 기능
- **블로그 포격**: 네이버, 구글 블로거 (긴 글 + 이미지)
- **SNS 침투**: 인스타그램, 스레드, X (짧은 문구 + 영상/이미지)
- **영상 타격**: 유튜브 쇼츠, 틱톡 (자동 조립 영상)
- **실시간 보고**: 모든 배포 결과는 DB(marketing_logs)에 자동 기록됩니다.

## 3. 설치 및 가동 방법 (단 2단계)
1.  **가져오기(Import)**: n8n 우측 상단 메뉴 ➔ `Import from File` ➔ `lily-marketing-v2.json` 선택
2.  **활성화(Activate)**: 우측 상단의 스위치를 **[ON]**으로 변경

## 4. 제로-컨피그(Zero-Config) 원리
본 엔진은 n8n 내부의 'Credentials' 설정을 사용하지 않습니다. LilyMag 앱 서버가 배포 시점에 사장님의 암호화된 API 키를 직접 노드에 주입하므로, 이동 및 복구가 매우 자유롭습니다.

## 5. 문제 해결 (Q&A)
- **배포가 안 돼요**: 앱의 `.env.local`에 `N8N_PUBLISH_WEBHOOK_URL`이 정확히 등록되었는지 확인하십시오.
- **영상이 안 만들어져요**: `Creatomate` 혹은 `Shotstack` API 엔드포인트가 정상인지 확인하십시오 (기본 세팅 완료).
- **결과를 어디서 보나요?**: 앱 대시보드의 '마케팅 리포트' 혹은 Supabase의 `marketing_logs` 테이블을 확인하십시오.
