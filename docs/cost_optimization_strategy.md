# [lilymagerp-v4] 유지비용 최소화 및 운영 효율화 전략 (v1.0)

## 💰 1. AI 비용 최적화 (AI Cost Management)
- **Model Tiering**: 단순 텍스트 생성은 `gpt-4o-mini`, 복잡한 이미지 분석/영상 기획은 `gpt-4o`를 사용하는 하이브리드 방식 채택.
- **Prompt Engineering**: 토큰 소모를 최소화하는 정밀한 프롬프트 설계로 1회 호출당 비용 절감.
- **Self-Caching**: 한 번 생성된 콘텐츠(추천 꽃말, 영상 대본 등)는 DB에 저장하여 동일한 요청 시 AI를 재호출하지 않음.

## 💾 2. 데이터베이스 및 스토리지 최적화 (DB & Storage)
- **Supabase 효율적 활용**: 무료 티어 내에서 최대한의 성능을 내도록 효율적인 관계형 DB 설계 (정규화 및 인덱싱).
- **Blob 압축**: 업로드되는 모든 사진은 WebP 형식으로, 영상은 H.265 코덱으로 클라이언트 단에서 1차 압축 후 서버 전송.
- **Lazy Loading**: 모든 데이터는 필요한 시점에만 불러와 트래픽 전송료 최소화.

## 🚀 3. 트래픽 및 컴퓨팅 비용 절감 (Traffic & Cloud)
- **Edge Computing**: 로직 실행을 서버가 아닌 Vercel Edge Functions에서 수행하여 서버 부하 및 실행 시간 비용 감소.
- **Static Site Generation (SSG)**: 변하지 않는 마케팅 페이지는 미리 빌드하여 불필요한 서버 연산 제거.
- **CDN Cache**: 전 세계 사용자에게 가장 가까운 서버에서 데이터를 즉시 응답하여 전송 속도는 높이고 비용은 낮춤.

## 🔗 4. 자동화 워크플로우 최적화 (Automation)
- **n8n 트리거 최적화**: 1분마다 체크하는 방식이 아닌, 이벤트 발생 시 즉시 실행되는 Webhook 방식으로 불필요한 실행 횟수 제거.
- **Batch Processing**: 비정기적인 마케팅 포스팅은 트래픽이 적은 시간에 모아서 처리하여 시스템 부하 관리.
